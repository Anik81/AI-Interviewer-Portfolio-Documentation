## 1. Screenshot Requirements

### Technical Specifications

**Image Format:**
- **Format**: PNG (preferred) or JPG
- **Resolution**: 1920x1080 minimum (Full HD)
- **Color Mode**: RGB
- **File Size**: < 2MB per image (compress if needed)

**Naming Convention:**
```
[section]-[feature]-[view]-[number].png

Examples:
- candidate-dashboard-overview-01.png
- hr-job-creation-form-01.png
- interview-live-interface-01.png
- results-detailed-view-01.png
```

### Image Quality Standards

- **Clarity**: Sharp, no blur or pixelation
- **Lighting**: Proper contrast, readable text
- **Composition**: Centered, well-framed
- **Content**: Relevant UI elements visible
- **Annotations**: Optional arrows/highlights for key features

## 3. Best Practices

### 3.1 Before Capturing

**Prepare the Environment:**
1. **Use Demo Data**: Create realistic but fake candidate information
   - Names: Use common placeholder names (John Doe, Jane Smith)
   - Emails: Use example.com domain
   - Phone numbers: Use fake numbers (555-0100 format)
   - Addresses: Use generic locations

2. **Clear Browser State**:
   - Clear console errors
   - Close unnecessary browser tabs
   - Hide bookmarks bar
   - Use clean browser profile

3. **Optimize Display**:
   - Close system notifications
   - Hide desktop icons (if capturing desktop)
   - Use consistent browser window size
   - Ensure proper zoom level (100%)

4. **Test Data Quality**:
   - Populate with meaningful data
   - Show various states (active, completed, pending)
   - Include realistic metrics

### 3.2 During Capture

**Screenshot Techniques:**

1. **Full Page Capture**:
   - Use browser extension for full-page scrolling screenshots
   - Recommended: Awesome Screenshot, Fireshot

2. **Focused Capture**:
   - Highlight specific features
   - Crop unnecessary chrome (if appropriate)
   - Maintain context

3. **Consistent Timing**:
   - Capture all screenshots in same session
   - Maintain UI consistency
   - Use same theme/settings

4. **Multiple Angles**:
   - Capture both overview and detail shots
   - Show interactions (hover states, dropdowns)
   - Include error states (if relevant)

### 3.3 After Capture

**Post-Processing:**

1. **Annotation** (Optional):
   - Add arrows to highlight key features
   - Use text callouts for complex features
   - Keep annotations minimal and professional
   - Tools: Snagit, Skitch, Photoshop

2. **Privacy Masking**:
   - Blur sensitive information
   - Replace real data with placeholders
   - Remove any company-specific branding (if required)

3. **Optimization**:
   - Compress images without quality loss
   - Tools: TinyPNG, ImageOptim, Squoosh
   - Target < 2MB per image

4. **Organization**:
   - Rename files according to convention
   - Sort into appropriate folders
   - Create backup copies

## 4. Privacy & Compliance

### 4.1 Data Protection Guidelines

**What to AVOID in Screenshots:**
- ❌ Real candidate personal information (names, emails, phone)
- ❌ Actual CV content with PII
- ❌ Real interview recordings or transcripts
- ❌ Production database data
- ❌ API keys, tokens, or credentials
- ❌ Internal company information
- ❌ Proprietary algorithms or business logic
- ❌ Real company logos (unless authorized)

**What is SAFE to Show:**
- ✅ UI layouts and design
- ✅ Anonymized demo data
- ✅ Generic placeholders
- ✅ Public-facing interfaces
- ✅ Sanitized metrics (approximations)
- ✅ Wireframes and mockups
- ✅ Feature interactions (with dummy data)

### 4.2 Company Policy Compliance

**Before Publishing:**
1. **Review Company Policy**: Check if company has social media/portfolio guidelines
2. **Get Approval**: Request written permission to share project details
3. **Use Demo Environment**: Always use staging/demo environment, not production
4. **Sanitize Data**: Triple-check for any sensitive information
5. **Watermark** (Optional): Add "Demo Data" watermark if needed

### 4.3 Sample Disclaimer

Include this disclaimer in your README:

```markdown
## Disclaimer

All screenshots and data shown in this documentation are from a demo/staging
environment created for portfolio purposes.

- All personal information is fictitious and generated for demonstration only
- Screenshots do not contain any real candidate data or company information
- Metrics and numbers are approximate and for illustrative purposes
- The actual production system may have additional features not shown here

This documentation has been reviewed and approved for public sharing by
[Company Name] in accordance with confidentiality agreements.
```

## 6. Tools & Software

### 6.1 Screenshot Capture Tools

**Browser Extensions:**
- **Awesome Screenshot** (Chrome, Firefox)
  - Full-page capture
  - Annotation tools
  - Cloud storage integration

- **Fireshot** (Chrome, Firefox)
  - PDF export
  - Entire page capture
  - Multiple formats

- **Nimbus Screenshot** (Chrome, Firefox, Edge)
  - Video recording
  - Screen recording
  - Built-in editor

**Desktop Applications:**
- **Snagit** (Windows, Mac) - Professional tool with advanced editing
- **Greenshot** (Windows, Free) - Open-source screenshot tool
- **Lightshot** (Windows, Mac) - Quick and simple
- **Skitch** (Mac) - Annotation-focused

**Built-in Tools:**
- **Windows**: Snipping Tool, Snip & Sketch (Win + Shift + S)
- **Mac**: Screenshot (Cmd + Shift + 3/4/5)
- **Linux**: Gnome Screenshot, Spectacle (KDE)

### 6.2 Image Editing Tools

**For Annotations:**
- **Photoshop** (paid) - Professional editing
- **GIMP** (free) - Open-source alternative
- **Canva** (freemium) - Online editor
- **Pixlr** (free) - Browser-based

**For Compression:**
- **TinyPNG** (online, free) - PNG/JPG compression
- **ImageOptim** (Mac, free) - Lossless compression
- **Squoosh** (online, free) - Google's image compressor
- **RIOT** (Windows, free) - Radical Image Optimization Tool

**For Mockups:**
- **Mockuper** (online) - Device mockup generator
- **Smartmockups** (online) - Professional mockups
- **Placeit** (online) - Quick mockup creation
- **Figma** (free) - Design tool with mockup templates

### 6.3 Browser Setup for Screenshots

**Recommended Browser Configuration:**

1. **Chrome/Edge**:
   ```
   - Resolution: 1920x1080 (full screen or specific size)
   - Zoom: 100%
   - Extensions: Disable unnecessary ones
   - DevTools: Open for responsive testing
   - Bookmarks: Hidden
   ```

2. **Firefox**:
   ```
   - Use Firefox Developer Edition for better DevTools
   - Responsive Design Mode (Ctrl + Shift + M)
   - Screenshot Tool (Ctrl + Shift + S) - built-in full page
   ```

**Useful DevTools Settings:**
- Disable cache
- Network throttling (to test loading states)
- Device emulation for responsive views
- Dark mode toggle
